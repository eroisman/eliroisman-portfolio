Docker

Premiers pas avec les conteneurs.
Dans cette page
A - Lancer un conteneur B - Lancer une tâche dans un conteneur

A - Lancer un conteneur :

Lancer un conteneur est très simple :

docker run <image qu’on veut lancer>:<version>

Dans docker, on lance des conteneurs à partir d’image : un conteneur est une instance d’une image. L’image est une recette, les conteneurs des gâteaux produits à partir de cette recette.

  1. Lancez la commande docker ps
    Cette commande permet de lister les conteneurs actifs (en fonctionnement).

    docker ps

    Lancez la commande docker ps -a
    Cette commande permet de lister tous les conteneurs (actifs et non-actifs)

    docker ps -a

    Normalement, pas grand chose ne devrait s’afficher : vous n’avez lancé aucun conteneur. Mais l’image que vous exploitez a normalement déjà lancer un conteneur...

  2. La commande docker images
    Cette commande liste toutes les images docker téléchargées sur votre système.
    Normalement, il n’y en a aucune. Mais, toujours pareil...comme la machine sur laquelle vous travaillez a déjà utilisé docker...

    docker images

  3. Lancez la commande docker run ubuntu:latest
    Cette commande lance un conteneur contenant un système Ubuntu dans la dernière version disponible de l’image.

    docker run ubuntu:latest

  4. Utilisez les commandes docker ps, docker ps -a et docker images

    docker ps -a
    docker image

  5. Utilisez la commande docker run --name deuxieme ubuntu:latest

    docker run --name deuxieme ubuntu:latest

    Lancez ensuite la commande docker ps -a et docker images

    docker ps -a
docker images

    Déduisez en l’utilité de l’option --name (à quoi elle sert) :

    L'option --name dans la commande docker run permet de donner un nom spécifique au conteneur qui est lancé. Dans l'exemple donné, la commande docker run --name deuxieme ubuntu:latest lance un nouveau conteneur à partir de l'image ubuntu:latest et lui donne le nom deuxieme.

B - Lancer une tâche dans un conteneur :

C’est très sympathique de lancer des conteneurs pour les voir inactifs, mais ça semble légèrement contre-productif… Apprenons à lancer un conteneur qui réalise une tâche !

  1. Lancez la commande docker run ubuntu:latest /bin/ls

    docker run ubuntu:latest /bin/ls

    On peut exécuter un programme spécifique à l'intérieur d'un conteneur en utilisant la commande docker run 'image' 'programme'. Dans cet exemple, la commande docker run ubuntu:latest /bin/ls exécute la commande ls à l'intérieur d'un conteneur basé sur l'image ubuntu:latest.

  2. Utilisez la commande docker ps puis docker ps -a

    docker ps
docker ps -a

  3. Utilisez la commande docker run ubuntu:latest /usr/bin/top -b
    L’option -b de top lance cette commande à intervalle de temps régulier. Elle peut être arrêtée avec Ctrl+c

    docker run ubuntu:latest /usr/bin/top -b

    Un conteneur fonctionne pendant la durée de l'exécution de la tâche qu'il doit effectuer puis il se termine.
    Lorsque nous utilisons la commande docker run ubuntu:latest /usr/bin/top -b, le conteneur basé sur l'image Ubuntu est démarré et exécute la commande /usr/bin/top -b à intervalle de temps régulier. Cependant, puisque nous n'avons pas spécifié d'action spécifique à exécuter, le conteneur exécutera la commande de manière continue jusqu'à ce qu'elle soit arrêtée manuellement. Dans ce cas, la commande top affiche les informations en temps réel sur les processus en cours d'exécution dans le conteneur.
